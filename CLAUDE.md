## Project Overview

**Travel Curator — Quotation System**: A full-stack internal tool for travel agencies to create, manage, and export professional travel quotations. Features a quotation builder, day-by-day itinerary, PDF export (client-facing, no prices), photo uploads, and an admin dashboard with bulk CSV service catalog import/export.

## Development Commands

```bash
npm install          # Install dependencies
cp .env.example .env # Create env file, then edit values
npm run dev          # Start with nodemon (auto-reload)
npm start            # Production start
npm run seed         # Seed sample data (4 destinations, 3 users, 2 quotations)
```

**No test runner or linter is configured.** Use `GET /health` to verify the server is running.

## Environment Variables

Minimum required in `.env`:
```
PORT=5000
MONGODB_URI=mongodb://localhost:27017/travel_quotations
JWT_SECRET=your_jwt_secret
```

## Architecture

**Request flow:** `routes/*.js` → `controllers/*.js` → `models/*.js` → MongoDB

```
Browser (Vanilla JS)
    ↓
Express server (server.js, port 5000)
    ├── Middleware: Helmet, rate-limit (100/15min), CORS
    ├── Static files: /public/
    └── 4 route groups:
        ├── /api/quotations  → quotationController.js
        ├── /api/destinations → destinationController.js
        ├── /api/admin        → adminController.js
        └── /api/users        → userController.js
    ↓
MongoDB (3 collections: quotations, destinations, users)
    Disk: /public/uploads/ (Multer, 10MB/image)
```

## Key Files

- `server.js` — Entry point: middleware stack, route mounts, global error handler
- `config/database.js` — MongoDB connection with graceful shutdown
- `models/Quotation.js` — Complex schema with hotel/service subdocs; pre-save hook auto-calculates checkout dates; `generateRefId()` creates format `QE-DU-09APR-1`; `calculateTotal()` sums hotels + services
- `models/Destination.js` — Embeds `cities[]`, `hotels[]`, and `cityServices[]` (the service catalog)
- `models/User.js` — bcrypt password hashing in pre-save hook; JWT-based auth via `comparePassword()`
- `controllers/adminController.js` — CSV parse/generate for bulk service catalog import/export and quotation export
- `public/js/app.js` (~1500 lines) — Quotation builder frontend
- `public/js/admin.js` (~700 lines) — Admin dashboard frontend

## Frontend Notes

No build process. Tailwind CSS is loaded via CDN. All frontend logic is vanilla JavaScript in `/public/js/`. Changes to frontend are immediately reflected — no compilation step needed.

## Data Model Highlights

- **Quotation** refId format: `QE-{DEST}-{DATE}-{SEQ}` (e.g. `QE-DU-09APR-1`). Auto-generated by `Quotation.generateRefId()` static method.
- **Destination** holds the service catalog as embedded `cityServices[]` array — this is what populates the Service Catalog Picker in the UI.
- **CSV import** (`POST /api/admin/import/catalog`) expects columns: `city, serviceType, serviceName, description, rate, currency`
- Photos are stored on disk and served as static files; paths are stored in the quotation document.
